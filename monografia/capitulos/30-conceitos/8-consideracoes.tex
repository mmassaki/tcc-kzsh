\section{Considerações finais do capítulo} % (fold)
\label{sec:considerações_finais_do_capítulo_2}

Este capítulo descreveu os principais conceitos teóricos e tecnologias
envolvidas neste projeto de formatura. A compreensão do conteúdo dos itens anteriores facilita o entendimento do desenvolvimento do projeto.

O \emph{iPhone} foi selecionado como plataforma de desenvolvimento devido ao seu grande número de usuários atualmente no Brasil, por fornecer ferramentas de desenvolvimento integradas e de uso simplificado e pelo interesse do grupo em aprofundar seus conhecimentos em desenvolvimento de aplicativos para \emph{iPhones}.

O uso do \emph{iPhone} como plataforma móvel permitiu o uso do sistema A-GPS de posicionamento, disponível nos aparelhos a partir da versão 3G, o uso do ambiente \emph{Xcode}, linguagem \emph{Objective-C} e \emph{Cocoa Touch}.

A implementação do servidor utilizando o \emph{framework} \emph{Ruby on Rails} permite um desenvolvimento mais fácil e ágil, além de ser uma tecnologia de conhecimento do grupo e, juntamente com a ferramenta \emph{Scaffolding} permite a criação mais eficiente de CRUDs e interfaces \emph{web services} em padrão \emph{RESTful}.

A \emph{Google Maps API} permite o acesso à informações geográficas necessárias para o mapeamento de trânsito e para a geração de rotas do sistema de forma gratuita e simples. Para o projeto adotamos a API via \emph{web services}, pois é a mais adequada para uso com o servidor, já que usa somente requisições HTTP.
