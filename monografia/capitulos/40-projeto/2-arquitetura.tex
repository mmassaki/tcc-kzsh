\section{Arquitetura do sistema} % (fold)
\label{sec:arquitetura_do_sistema}

A arquitetura geral do sistema, incluindo todos os seus componentes, é apresentada na figura \ref{fig:arquitetura}.

\begin{figure}[htpb]
	\centering
	\includegraphics[scale=0.5]{img/arquitetura.png}
	\caption{Arquitetura do sistema}
	\label{fig:arquitetura}
\end{figure}

A arquitetura está dividida basicamente em duas camadas físicas: 

\textbf{Servidor de aplicação e banco dados:} Responsável por armazenar e disponibilizar os dados persistentes enviados pelos usuários do \emph{BetterWay} via internet e utilizados pelo sistema de trânsito. A partir dos dados, manipula a requisição com a \emph{API Google Maps} para gerar uma melhor rota para o dispositivo com o aplicativo. Também fornece o resultado da manipulação dos dados para a exibição da malha do tráfego atual.

\textbf{Cliente (aplicação \emph{BetterWay}):} Nó referente aos usuários acessando e utilizando a aplicação \emph{BetterWay}. Na aplicação está a interface que apresenta o mapa com todas informações disponibilizadas pelo servidor. Possui um sistema interno para gerir endereços recentes buscados, favoritos e também de informar eventos.
