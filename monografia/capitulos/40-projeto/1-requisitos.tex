\section{Especificação de requisitos} % (fold)
\label{sec:especificação_de_requisitos}

Esta seção apresenta resumidamente as especificações de requisitos do sistema desenvolvido. A especificação completa encontra-se no Apêndice~\ref{cha:especificação_de_requisitos_do_sistema}.

\subsection{Requisitos funcionais} % (fold)
\label{sub:requisitos_funcionais}

As funções especificadas do sistema são:

\textbf{Navegar no mapa:} o sistema deve permitir que o usuário veja e possa navegar em um mapa de ruas.

\textbf{Localizar posição atual:} o sistema deve informar a posição atual do usuário

\textbf{Verificar trânsito:} o sistema deve exibir informações de trânsito.

\textbf{Traçar rota:} o sistema deve ser capaz de traçar uma rota para o usuário.

\textbf{Seguir rota:} o sistema deve permitir que o usuário siga uma rota previamente traçada.

\textbf{Enviar registros:} o sistema deve coletar informações de velocidade dos usuários enviando-as para o servidor, que deve cadastrá-las no banco de dados.

\textbf{Mapear trânsito:} o sistema deve, a partir das informações de registros armazenadas, gerar informações de trânsito das vias que os usuários percorreram e armazená-las no servidor.

\textbf{Informar eventos:} o sistema deve permitir que os usuário enviem eventos relacionados ao trânsito afim de compartilhar informações. Para isso o servidor terá de cadastrar e permitir a consulta dessas informações.

\textbf{Temporização dos eventos e informações de trânsito:} as informações dos eventos e trânsito devem expirar após 30 minutos caso não sejam atualizadas por nenhum usuário.

\textbf{Enviar destino:} o sistema deve permitir que o usuário envie a outras pessoas o seu local de destino.

\textbf{Salvar e remover favoritos:} o sistema deve permitir que o usuário armazene e remova seus locais favoritos.

% subsection requisitos_funcionais (end)

\subsection{Requisitos não funcionais} % (fold)
\label{sub:requisitos_não_funcionais}

Os principais requisitos não funcionais que o sistema deve atender são:

\textbf{Usabilidade:} é essencial que o aplicativo possua uma interface com o usuário que exija a menor interação possível, uma vez que pode ser utilizado diretamente por motoristas.

\textbf{Desempenho:} o aplicativo deve responder aos estímulos do usuário rapidamente, bem como realizar todas as suas funções, como envio de mensagens, auxílio de navegação e informação de tráfego, de modo eficiente, para que a atenção do motorista não seja desviada.
