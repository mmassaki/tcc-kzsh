\subsection{Escopo}
A aplicação, específica para a plataforma iPhone, permite que o usuário se localize em um mapa com informações de trânsito. Será configurado um servidor para que haja comunicação entre o aplicativo e um banco de dados com informações históricas adquiridas.

\section{Descrição geral}
Trata-se de um sistema de navegação veicular para a plataforma iPhone. São utilizados os recurso de posicionamento global - GPS - e de conexão à internet, para que o usuário possa se localizar em um mapa e obter informações do trânsito em tempo real.

As informações das condições de tráfego serão obtidas dos próprios usuários da aplicação, de forma automática, independente da ação direta deles e serão concentradas e armazenadas em um servidor dedicado. Elas também são disponibilizadas aos usuários, que poderão visualizá-las na forma de um mapa com uma escala de cores, ou seja, as ruas serão coloridas, variando do vermelho ao verde, indicando condições de congestionamento até tráfego livre.

A aplicação, portanto, se comunica pela internet com o servidor para obter as informações históricas e atuais pertinentes à região em que o usuário se encontra, que é determinada pelo GPS do aparelho.

Como as condições são atualizadas em tempo real, o sistema também é capaz de recalcular a rota autonomamente, caso haja mudanças relevantes no tráfego.

		\subsubsection{Mapas}
		Disponibilizar um mapa atualizado que sirva de interface ao usuário.

		\subsubsection{Navegação}
		\begin{itemize}
			\item{\bf Trânsito}

			Ao trafegar pelas vias, a aplicação envia automaticamente a condição atual do trânsito para o servidor, que é calculada por meio da localização e da velocidade do usuário, obtidas por meio do recurso de GPS. Essa informação é disponibilizada para os outros usuários que também estão utilizando a aplicação.

			\item{\bf Mapeamento em tempo real}

			As informações do trânsito são exibidas automaticamente por meio de coloração, que segue uma escala de trânsito livre, intenso, lento ou congestionado.

			\item{\bf Localização atual}

			A aplicação apresenta a localização no mapa em tempo real, mesmo se não estiver seguindo alguma rota. É exibido um ícone, indicando sua posição.
			
			\item{\bf Alertas}
			
			A aplicação permite que o usuário envie alertas de eventos relacionados ao trânsito, como semáforos quebrados, alagamentos, acidentes e outros.
			
			\item{\bf Rotas}
			
			O sistema é capaz de, dadas duas localizações, calcular uma rota entre elas. Se o usuário decidir seguir uma rota, basta informar ao sistema para que ele passe a funcionar como um navegador, indicando graficamente o caminho a ser seguido.
		\end{itemize}

		\subsubsection{Servidor}
		\begin{itemize}
			\item{\bf Manipulação de banco de dados}

			Registrar e acessar, no banco de dados, informações de condições de tráfego das vias e alertas emitidos.

			\item{\bf Geração e consulta de estatísticas}

			Tratar dados históricos registrados no banco de dados, para auxiliar no roteamento do usuário.
		\end{itemize}
		
		\subsubsection{Integração}
		O sistema é capaz de enviar mensagens de alertas para o Twitter e para contatos do usuário, presentes na agenda telefônica do iPhone, por meio de SMS e e-mail.

\section{Requisitos específicos}

	\subsection{Modelo de casos de uso}
	Para a modelagem de casos de uso definiu-se como sistema o dispositivo iPhone juntamente com o Servidor de dados.
	
	O aplicativo desenvolvido para o iPhone, após sua inicialização, exibe a tela principal com o mapa do sistema.
	
	Para o funcionamento da aplicação é necessária uma rede de dados acessada através do iPhone. Caso nenhuma rede esteja disponível a aplicação informará o usuário que não foi possível conectar-se em nenhuma rede de dados.
	
	Para que a aplicação obtenha a informação de posição do dispositivo é necessário que o usuário permita o acesso ao GPS para aplicação. Caso a aplicação não tenha essa permissão uma mensagem será informada ao usuário para que ative-a, gerando uma extensão ao caso de uso.
	
	Devido à interface do iPhone, toda interação com o usuário é feita através de botões, toques ou gestos na tela.

		\textbf{Caso de uso 16:} Enviar via Facebook

	 	\textbf{Descrição:} Este caso de uso descreve o envio do local de destino via Facebook.

		\textbf{Evento iniciador:} Solicitação de envio do local de destino via Facebook.

		\textbf{Atores:} Usuário

		\textbf{Pré-condição:} Sistema na tela de seleção de meio de envio de destino.

		\textbf{Seqüência de eventos:}
		\begin{enumerate}
			\item Sistema exibe a mensagem a ser enviada podendo ser editada;
			\item Usuário edita a mensagem se necessário e confirma o envio;
			\item Sistema envia a mensagem.
		\end{enumerate}

		\textbf{Pós-condição:} Mensagem via Facebook enviada.

		\textbf{Extensões:}
		\begin{enumerate}
			\item Usuário não autenticado no Facebook (passo 3): Sistema solicita a autenticação do usuário no Facebook.
		\end{enumerate}

		\textbf{Exceções:}
		\begin{enumerate}
			\item Erro ao enviar mensagem via Facebook (passo 3): sistema exibe mensagem ao usuário.
		\end{enumerate}

\section{Requisitos não funcionais}

\begin{itemize}
	\item {\bf Usabilidade}
	
	É essencial que o aplicativo possua uma interface com o usuário que exija a menor interação possível, uma vez que pode ser utilizado diretamente por motoristas.
	
	\item {\bf Desempenho}
	
	O aplicativo deve responder aos estímulos do usuário rapidamente, bem como realizar todas as suas funções, como envio de mensagens, auxílio de navegação e informação de tráfego, de modo eficiente, para que a atenção do motorista não seja desviada.	
	
\end{itemize}
