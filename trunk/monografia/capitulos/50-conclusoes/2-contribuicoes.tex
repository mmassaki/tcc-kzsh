\section{Contribuições} % (fold)
\label{sec:contribuições}

A monografia e a parte prática desenvolvida neste projeto pretendem deixar uma contribuição futura para os projetos destinados a aplicativos móveis voltados a navegação terrestre.

Atualmente não há comercialmente um aplicativo de navegação com informações de trânsito atualizadas constantemente que use informações disponibilizadas pelos usuários. O aplicativo mais próximo encontrado é o \emph{Waze} que permite que seus usuários informem eventos de trânsito, mas possui um mapa extremamente desatualizado da cidade de São Paulo. O sistema desenvolvido visa estimular a criação de um sistema comercialmente viável baseado nestes mesmos princípios.

O desenvolvimento do trabalho possibilitou principalmente, como contribuição para os integrantes do grupo, o aprendizado de diferentes linguagens de programação, além de aplicações de metodologias aprendidas no decorrer do curso de Engenharia de Computação.

Além disso, foi necessário um grande esforço na parte de pesquisas para o desenvolvimento do sistema, capacitando os membros do grupo para esta área tão importante na carreira de um engenheiro de computação.

O conhecimento adquirido durante o processo de pesquisa e desenvolvimento do projeto foram principalmente:

\textbf{GPS:} o estudo do funcionamento do sistema global de posicionamento permitiu o entendimento de falhas de obtenção do sinal em locais fechados.

\textbf{Aprendizado na plataforma de desenvolvimento da Apple, o \emph{Xcode}, utilizando a linguagem \emph{Objective-C} com a arquitetura  \emph{Cocoa Touch}:}

\textbf{\emph{RESTful}:} o uso do padrão \emph{RESTful} exigiu que fossem estudados os métodos de acesso ao recurso disponibiizado via \emph{web services} para customização dos métodos da classe.

\textbf{\emph{Google Maps API}:} as APIs fornecidas pelo \emph{Google Maps} foram fundamentais para execução das funcionalidades do sistema. O estudo do funcionamento destas interfaces permitiram tirar melhor proveito destes serviços.

\textbf{Utilização de técnicas de Engenharia de Software:} foram utilizadas as técnicas aprendidas em sala nas aulas de Engenharia de Software, tanto em documentação, como na modelagem de casos de uso, classes, estados, interfaces, etc.

\textbf{Utilização de técnicas de gerenciamento de configuração:} foram utilizadas técnicas de gerenciamento de configuração vistas na disciplina de Gerência e Qualidade de Software para o desenvolvimento de componentes e documentos, como a criação de versões, \emph{baselines}, cronograma, repositório, etc.
